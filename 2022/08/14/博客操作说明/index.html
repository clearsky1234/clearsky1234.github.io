博客操作说明

本文最后更新于：2022年8月14日 晚上

创建md文件

创建一个新的页面，使用如下命令

$ hexo new <br>$ hexo new “我的第一篇文章”</p> </blockquote> <h2 id="布局"><a href="#布局" class="headerlink" title="布局"></a>布局</h2><p>在创作博客的时候可以指定布局<br>语法如下</p> <blockquote> <p>$ hexo new [layout] <title><br>$ hexo new page “我的页面”</p> </blockquote> <p>布局分为post（文章）、draft（草稿）、page（页面）这三种，可以理解为是hexo对不同博文的分类，hexo会把他存在相应的文件夹下。<br>在默认情况下，创建的都是post，即简单的一个文章，这也是最常用的。当创建一个页面的时候，hexo会创建一个以标题为名字的目录，里面包含一个index.md的文件。创建草稿的时候，文章是不会在网页上展示的，如果想把该文章发布，就可以使用以下命令<br>hexo publish [layout] <title></p> <h2 id="文章分类和标签"><a href="#文章分类和标签" class="headerlink" title="文章分类和标签"></a>文章分类和标签</h2><p>在每个新建md文档当中，头上会有一个用——————分割的区域，这里被称为Front-matter<br>这里的预定义的参数含义如下</p> <blockquote> <p>layout 布局 默认为true，如果你不想你的文章被处理，可以设置为false<br>title 标题 标题会显示在最上方居中位置<br>date 建立日期 如果不指定则为默认值-文件创建日期，可以自定义。<br>update 更新日期 如果不指定则为默认值-文件修改后重新生成静态文件的日期。<br>comments 是否开启文章的评论功能 默认值为true<br>tags 标签（不适用于页面page布局）<br>categoreies 分类（不适用于页面page布局）<br>permalink 覆盖文章网址<br>keywords 仅用于 meta 标签和 Open Graph 的关键词（不推荐使用）</p> </blockquote> <p>在这其中我们最常用到就是分类（categories）和标签（tags）了，这两者的区别就是分类可以有层级，标签没有<br>使用例子如下：</p> <blockquote> <p>categories:</p> <ul> <li>个人博客（第一层级）</li> <li>Hexo博客（第二层级）<br>tags:</li> <li>Hexo</li> <li>博客<br>多分类的例子如下：<br>categories:</li> <li>[日常, 生活]</li> <li>[日常, 随想]</li> <li>[日记]</li> </ul> </blockquote> <h2 id="部署与发布"><a href="#部署与发布" class="headerlink" title="部署与发布"></a>部署与发布</h2><p>清除缓存：hexo clean<br>生成静态文件：hexo generate可简写为 hexo g<br>启动服务器：hexo server或者 hexo s 常用参数：-p（–port）重设端口<br>部署：hexo deploy可简写为hexo d，用于将网站部署到服务器上。（暂时用不到，目前都是在本地，后面我们将博客托管到GitHub Pages或Gitee Pages时才会用到此命令）<br>常用参数：-g（–generate），hexo d -g部署前预先生成静态文件，等同于 hexo g -d<br>命令如下：</p> <blockquote> <p>$ hexo clean && hexo g && hexo s<br>$
